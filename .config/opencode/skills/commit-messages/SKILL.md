## Format
Always use: `<type>: <short summary>`

Types: feat, fix, docs, style, refactor, test, chore

## Rules
- Subject line max 100 chars, lowercase, no period, no new line
- Use past tense mood: "added feature" not "add feature"
- Body explains *what*, sometimes *why*
- Reference issues: `Closes #123`

## Examples
- feat: added OAuth2 login flow
- fix: handle null response from payment gateway
- refactor: extract query builder into separate module
